PrgLnch Config

Shortcut Slots: Select a Shortcut Slot to fill or modify it, or clear the text in order to remove it with the del key or Modify Shortcut button below. Else leave at "None" to change the resolution of the selected monitor. The Del key is reserved for removing all contents of the box.
Modify Shortcut: Depending on the contents of Shortcut Slot above, the current Prg name and/or path can be added, modified, or removed with this button.
Command Line Extras: Paste in any extra parameters or arguments required for the Prg to run. Not available if the Prg is a shortcut. The Del key removes everything in the box.
Monitor Name: This displays the system name of the monitor that the current Prg will run on. Blue indicates a primary monitor, orange indicates a secondary monitor, and the grey, spare monitor slots,
Monitor List: Numbered list of monitors. The colour of the MonitorName above indicates its usabilty.
Show At Startup: If checked, the current Prg shows as selected at next PrgLnch run.

Res Options

TestMode:
System tests if the resolution works and PrgLnch notifies on success or fail.
Change at every mode:
Resolution is changed as in Dynamic below even if the target resolution matches the current resolution. PrgLnch will advise, even if this option is not chosen.
Dynamic: (All running Apps):
Resolution change is permanent. All apps running will receive this change and may or may not resize accordingly. The position of the items on the desktop will change.
Temporary (Recommended):
Resolution changes are in effect only while PrgLnch is running.

Pull values from registry

Retrieves the settings for the monitor that are currently stored in the registry rather than those used for the current monitor before changing resolution. In most cases they will match.
Default Resolution: Currently used (or default) resolution for the selected monitor.
List All Compatible: Checking this box expands the list below to all available resolution modes for the selected monitor.
Resolution Modes: List of resolution modes for the selected monitor. 32 bit bits per pixel (True Colour) is the current standard.
Change Resolution on Switch: Advanced Users: If a Prg is run on the same monitor as PrgLnch at a resolution other than the Default Resolution, switching from Prg to PrgLnch and vice-versa will change the resolutions accordingly.
Hide PrgLnch on Run: Hides the PrgLnch Gui after a Prg is launched. The Gui will re-appear when either the Prg is terminated, or is active in a Batch Preset followed by other Prgs launched in the Preset that do not have this setting checked.
Borderless: Applies the Borderless Attribute to the Prgs Window. Can be toggled while the Prg is active.
Priority: Default value is N (Normal). Check the box for H (High) priority or clear it for BN (Below Normal). Can be toggled while the Prg is active.
Prg Can Be a Shortcut to File: When checked, Prg can be a file with a *.lnk extension. This happens when selecting a .lnk file in the Modify Shortcut dialog. Otherwise .lnk targets are resolved.
Apply LAA Flag: Writes an Large Address Aware flag to the executable. Most Prgs have this setting already.
Test Run Prg: Either runs the Prg or cancels one that is running already. A notification is generated if the Prg is already active in a Batch Preset.
Update Prg: Click this to update Prg to a newer version1 at the URL Location displayed in the box underneath. A progress bar and Prg Update Status indicate the download status, and on completion of the download a prompt will appear for the Prg to be launched. If Prg is a .lnk file, the Update Prg button is unavailable. Although executable files at remote URLs are better handled by PrgLnch, compressed files require external decompression programs before use.
Url Name: Type in a valid Url of where the newer version of Prg is expected to reside. The Del key deletes all contents of the edit box
Prg Update Status: A brief informational message relating to the version and availablity of Prg at the above Url.
Back to Prglnch: All values are saved and returns to the PrgLnch Batch dialog.

Author reference: Version checking is done as described here: 


UrlDownloadToFile,
https://autohotkey.com/download/1.1/version.txt, C:\AutoHotkey Latest Version.txt
UrlDownloadToFile, http://someorg.org/archive.zip, C:\SomeOrg's Archive.zip

; Example: Download text to a variable:
whr := ComObjCreate("WinHttp.WinHttpRequest.5.1")
whr.Open("GET", "https://autohotkey.com/download/1.1/version.txt", true)
whr.Send()
; Using 'true' above and the call below allows the script to remain responsive.
whr.WaitForResponse()
version := whr.ResponseText
MsgBox % version
